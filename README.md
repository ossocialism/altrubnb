Airbnb and similar techno-lords (see [**Technofeudalism**](https://simple.wikipedia.org/wiki/Technofeudalism)) innovated by doing something genuinely new at the time:

They built large scale SaaS (Software as a service) platforms, solving the engineering challenges of the time to successfully **coordinate millions of decentralized service providers and and customers via a single centralized marketplace**.

But today, the SaaS software architecture ‚Äúmoat‚Äù is gone.

- building SaaS platforms is no longer an engineering miracle
- modern tooling and AI make platform development widely accessible
- open-source products have a proven track record (The best example ironically, is Kubernetes, a software infrastructure technology that allows corporations to run large scale SaaS platforms optimally).

There is no good reason municipalities and governments cannot compete when:

- the platform is open source and developed by a large community
- SaaS Infrastructure and Architecture is de facto standardized

Open source means:

- municipalities from all over world can collaborate
- municipalities don‚Äôt beg platforms for data ‚Äî they own the system
- each municipality is free to run the service on their own terms so
    - communities can shape rules that fit local housing realities
    - the economic value created locally **stays local**

This is the same pattern that played out with Kubernetes:

- multiple interested parties shared a common problem
- open source infrastructure beat proprietary silos
- collaboration replaced dependence on a single vendor

Housing platforms are ready for the same transformation.

### That‚Äôs how the cookie crumbles

Entrepreneurs and innovators break new ground, reap the early rewards of their ideas, and for a brief moment enjoy the unique fruits of being first. The same way Nespresso turned capsule coffee into a premium experience and later saw supermarket shelves fill with near-identical pods. In every cycle the pattern repeats: invention, harvest, imitation, diffusion.
